Generate plain-English documentation for a file or module.

The user will provide a file path. Read the file, then write documentation covering:

- **What it does** — one sentence summary
- **Why it exists** — what problem it solves
- **Key classes / functions** — what each one does, inputs and outputs
- **How to use it** — a short example
- **Things to watch out for** — any gotchas or important notes

Write it clearly enough that a new developer joining the project could understand it immediately.
